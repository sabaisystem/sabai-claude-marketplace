# /storyboard

Turn a video brief into a storyboard for a Remotion build.

## Usage

```
/storyboard [brief]
```

## When to Use

- The user wants scenes, beats, and timings before implementation
- The user has a script and needs visual treatment
- The user needs to review pacing with teammates

## Instructions

1. Identify the video's narrative arc:
   - Hook
   - Context
   - Main beats
   - CTA or ending
2. Split the concept into scenes with realistic durations.
3. For each scene, specify:
   - On-screen content
   - Spoken or written copy
   - Motion direction
   - Transition in/out
4. Call out scenes that should be reusable across aspect ratios.
5. If the storyboard feels too dense for the requested duration, say so and compress it.

## Output Format

```markdown
# Storyboard: [Title]

| Scene | Duration | Visual | Copy | Motion | Transition |
|-------|----------|--------|------|--------|------------|

## Pacing Notes

## Reusable Elements

## Risks / Open Questions
```
