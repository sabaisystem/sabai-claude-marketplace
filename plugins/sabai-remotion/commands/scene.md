# /scene

Design or refine a single scene for a Remotion composition.

## Usage

```
/scene [scene goal or draft]
```

## When to Use

- One scene is harder than the rest and needs focused work
- The user wants better visual hierarchy or timing
- The user needs copy, layout, and animation details for a single beat

## Instructions

1. Identify the scene's purpose in the larger video.
2. Define the scene using:
   - Duration
   - Layout
   - On-screen elements
   - Copy or narration
   - Animation timing
   - Entry and exit transitions
3. Favor one clear motion idea over multiple competing animations.
4. If the user shares existing component names or code, map the scene to them directly.
5. End with implementation notes that a developer can translate into a Remotion component.

## Output Format

```markdown
# Scene Spec: [Scene Name]

## Purpose

## Timeline
- 0%:
- 25%:
- 50%:
- 75%:
- 100%:

## Layout

## Copy

## Animation Notes

## Remotion Implementation Notes
```
