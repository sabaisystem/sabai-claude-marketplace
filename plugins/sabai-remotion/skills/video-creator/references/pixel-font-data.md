# 5×7 Pixel Font Reference

Character map for the 3D Retro Pixel Font template. Each character is defined on a 5-wide × 7-tall grid.

## Encoding Format

Each character is 7 rows of 5-character binary strings: `1` = filled pixel, `0` = empty.

```
"A": ["01110","10001","10001","11111","10001","10001","10001"]
```

Renders as:
```
 XXX
X   X
X   X
XXXXX
X   X
X   X
X   X
```

## Adding New Characters

To add a new character:

1. Design it on a 5×7 grid (graph paper or text editor)
2. Convert each row to a 5-character binary string (`1` = filled, `0` = empty)
3. Add it to the `FONT` object in the template code
4. Update this reference file with the visual grid

Example — adding `#`:
```
 X X
 X X
XXXXX
 X X
XXXXX
 X X
 X X
```
→ `["01010","01010","11111","01010","11111","01010","01010"]`

## Layout Rules

- Character width: 5 pixels
- Character height: 7 pixels
- Gap between characters: 1 pixel
- Gap between lines: 2 pixels
- Line width for N characters: `N × 6 - 1` pixels
- Total height for M lines: `M × 9 - 2` pixels
