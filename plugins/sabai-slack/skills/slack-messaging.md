# Slack Messaging Best Practices

Guidelines for effective Slack communication through Claude.

## Core Principles

### Be Concise
- Keep messages brief and scannable
- Use bullet points for multiple items
- Lead with the key information

### Use Threads Appropriately
- Reply in threads to keep channels organized
- Start new threads for new topics
- Use channel messages for announcements

### Choose the Right Channel
- Use public channels for team visibility
- Use DMs for sensitive or personal matters
- Use group DMs for small, focused discussions

## Message Formatting

### Text Formatting
```
*bold* for emphasis
_italic_ for subtle emphasis
~strikethrough~ for corrections
`code` for inline code or commands
```

### Block Formatting
```
> Blockquote for quoting others

```code block```
for multi-line code

• Bullet points with bullet character
1. Numbered lists
```

### Links and Mentions
- `@username` to notify someone
- `@channel` to notify everyone (use sparingly)
- `@here` to notify active members
- `<url|display text>` for formatted links

## Communication Patterns

### Status Updates
```
Update: [Brief summary]
- Key point 1
- Key point 2
Next steps: [Action items]
```

### Questions
```
Question: [Clear, specific question]
Context: [Relevant background]
Options considered: [If applicable]
```

### Requests
```
Request: [What you need]
Deadline: [When you need it]
Context: [Why it matters]
```

## Threading Etiquette

### When to Use Threads
- Follow-up questions on a topic
- Detailed discussions that don't need channel visibility
- Responses to announcements
- Debugging or troubleshooting conversations

### When to Post in Channel
- New announcements or updates
- Time-sensitive information
- Topics requiring broad input
- Final decisions or outcomes

## Timing Considerations

### Before Sending
- Check recent messages for context
- Verify you're in the right channel
- Consider if a thread reply is more appropriate

### Message Timing
- Avoid sending during off-hours unless urgent
- Use scheduled messages for non-urgent items
- Consider time zones for global teams

## Tips for Clarity

- State your ask upfront
- Provide necessary context, but don't over-explain
- Use formatting to improve readability
- Include relevant links or references
- Tag people only when their attention is needed
